### 2.2 架构分层说明

| 层级 | 职责 | 技术组件 |
|------|------|----------|
| **客户端层** | 用户界面与交互 | Vue3 + Vite + Element Plus + Axios + Pinia |
| **网关层** | 静态资源、反向代理、负载均衡 | Nginx |
| **控制器层** | 请求参数校验、路由分发、统一返回 | Spring MVC `@RestController` |
| **业务服务层** | 业务逻辑、事务管理、缓存协调 | Spring `@Service` + `@Transactional` |
| **数据访问层** | 数据库操作、SQL映射 | MyBatis-Plus `BaseMapper` |
| **安全层** | 身份认证、权限校验 | JWT Filter + `@RequireRole` AOP |
| **缓存层** | 数据缓存、分布式锁、限流计数 | Redis + RedisUtil |
| **存储层** | 业务数据持久化、文件存储 | MySQL 8.0 + 本地文件系统 |

## 4. 数据库设计

### 4.1 设计规范

#### 4.1.1 表前缀分组

按业务模块统一表前缀，区分 **系统权限表** 与 **业务资产表** 两大类别：

| 类别 | 前缀 | 模块 | 说明 |
|------|------|------|------|
| **系统权限** | `sys_` | 用户/角色/部门/菜单/字典/配置 | RBAC权限底座，独立于业务 |
| **系统日志** | `sys_` | 操作日志/登录日志 | 审计追踪，与权限表同前缀 |
| **资产台账** | `asset_` | 资产信息/附件/折旧 | 资产核心主数据 |
| **采购入库** | `proc_` | 采购单/供应商 | 采购入库业务 |
| **领用归还** | `req_` | 领用单/审批日志 | 领用归还业务 |
| **资产调拨** | `trans_` | 调拨单 | 调拨业务 |
| **维保报修** | `repair_` | 报修单/维修记录 | 维保业务 |
| **报废处置** | `scrap_` | 报废单 | 报废业务 |
| **盘点管理** | `inv_` | 盘点任务/明细/差异 | 盘点业务 |
| **AI查询** | `ai_` | AI查询日志 | AI调用日志 |

#### 4.1.2 通用审计字段

所有业务表统一包含以下 5 个审计字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| `create_time` | DATETIME | 创建时间，DEFAULT CURRENT_TIMESTAMP |
| `create_by` | VARCHAR(50) | 创建人用户名 |
| `update_time` | DATETIME | 修改时间，ON UPDATE CURRENT_TIMESTAMP |
| `update_by` | VARCHAR(50) | 修改人用户名 |
| `is_deleted` | TINYINT(1) | 删除标记：0-正常，1-已删除，DEFAULT 0 |

#### 4.1.3 命名规范

| 规范项 | 规则 | 示例 |
|--------|------|------|
| 表名 | `{前缀}_{业务名}`，全小写下划线 | `sys_user`, `asset_info`, `req_order` |
| 主键 | `id` BIGINT UNSIGNED AUTO_INCREMENT | — |
| 字段名 | 小写下划线，动词在前 | `expect_return_date`, `is_deleted` |
| 逻辑外键 | `{关联表名}_id`，COMMENT标注 `FK → {目标表}.id` | `dept_id` COMMENT '所属部门ID (FK → sys_dept.id)' |
| 索引名 | `idx_{表名}_{字段名}` | `idx_asset_info_asset_code` |
| 唯一索引 | `uk_{表名}_{字段名}` | `uk_sys_user_username` |
| 物理外键 | **禁用**，所有参照完整性由应用层保证 | — |
| 字符集 | `utf8mb4`，排序规则 `utf8mb4_general_ci` | — |
| 引擎 | InnoDB | — |
| 业务编号 | `*_no` 格式，独立字段非主键 | `asset_code`, `apply_no` |

#### 4.1.4 主键策略

所有表主键使用 MyBatis-Plus `IdType.AUTO`（数据库自增 BIGINT UNSIGNED）。业务编号（如 `AS-IT-2606-0001`、`RY-20260624-0001`）为独立字段，在 Service 层通过 Redis 分布式锁保护流水号生成。

### 4.3 完整建表SQL

#### 4.3.1 系统权限表

```sql
-- =====================================================
-- 1. 用户表 (sys_user)
-- 用途: 存储所有系统登录用户账号信息
-- =====================================================
CREATE TABLE `sys_user` (
    `id` BIGINT UNSIGNED NOT NULL AUTO_INCREMENT COMMENT '主键ID',
    `username` VARCHAR(50) NOT NULL COMMENT '登录账号，全局唯一',
    `password` VARCHAR(200) NOT NULL COMMENT '登录密码（BCrypt加密存储）',
    `real_name` VARCHAR(50) NOT NULL COMMENT '员工真实姓名',
    `dept_id` BIGINT UNSIGNED DEFAULT NULL COMMENT '所属部门ID (FK → sys_dept.id)',
    `phone` VARCHAR(20) DEFAULT NULL COMMENT '手机号（11位，全局唯一校验）',
    `email` VARCHAR(100) DEFAULT NULL COMMENT '邮箱地址（全局唯一校验）',
    `status` TINYINT UNSIGNED NOT NULL DEFAULT 1 COMMENT '账号状态: 0-禁用(不可登录), 1-启用',
    `last_login_time` DATETIME DEFAULT NULL COMMENT '最近一次登录成功时间',
    `last_login_ip` VARCHAR(50) DEFAULT NULL COMMENT '最近一次登录来源IP地址',
    `create_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '记录创建时间',
    `create_by` VARCHAR(50) DEFAULT NULL COMMENT '创建人用户名',
    `update_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '记录最后修改时间',
    `update_by` VARCHAR(50) DEFAULT NULL COMMENT '最后修改人用户名',
    `is_deleted` TINYINT UNSIGNED NOT NULL DEFAULT 0 COMMENT '软删除标记: 0-正常, 1-已删除',
    PRIMARY KEY (`id`),
    UNIQUE KEY `uk_sys_user_username` (`username`),
    KEY `idx_sys_user_dept_id` (`dept_id`),
    KEY `idx_sys_user_phone` (`phone`),
    KEY `idx_sys_user_status` (`status`),
    KEY `idx_sys_user_real_name` (`real_name`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_general_ci COMMENT='系统用户表';

-- =====================================================
-- 2. 角色表 (sys_role)
-- 用途: 定义系统角色及角色编码，为RBAC权限底座核心
-- =====================================================
CREATE TABLE `sys_role` (
    `id` BIGINT UNSIGNED NOT NULL AUTO_INCREMENT COMMENT '主键ID',
    `role_name` VARCHAR(50) NOT NULL COMMENT '角色名称（中文展示）',
    `role_code` VARCHAR(50) NOT NULL COMMENT '角色编码: ROLE_SUPER_ADMIN/ROLE_ASSET_ADMIN/ROLE_DEPT_ADMIN/ROLE_EMPLOYEE',
    `description` VARCHAR(200) DEFAULT NULL COMMENT '角色职责描述',
    `is_system` TINYINT UNSIGNED NOT NULL DEFAULT 0 COMMENT '是否系统预置角色: 0-自定义, 1-系统预置(不可删除)',
    `create_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '记录创建时间',
    `create_by` VARCHAR(50) DEFAULT NULL COMMENT '创建人用户名',
    `update_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '记录最后修改时间',
    `update_by` VARCHAR(50) DEFAULT NULL COMMENT '最后修改人用户名',
    `is_deleted` TINYINT UNSIGNED NOT NULL DEFAULT 0 COMMENT '软删除标记: 0-正常, 1-已删除',
    PRIMARY KEY (`id`),
    UNIQUE KEY `uk_sys_role_role_code` (`role_code`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_general_ci COMMENT='系统角色表';

-- =====================================================
-- 3. 用户-角色关联表 (sys_user_role)
-- 用途: N:N关联——一个用户可拥有多个角色
-- =====================================================
CREATE TABLE `sys_user_role` (
    `id` BIGINT UNSIGNED NOT NULL AUTO_INCREMENT COMMENT '主键ID',
    `user_id` BIGINT UNSIGNED NOT NULL COMMENT '用户ID (FK → sys_user.id)',
    `role_id` BIGINT UNSIGNED NOT NULL COMMENT '角色ID (FK → sys_role.id)',
    `create_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '关联创建时间',
    PRIMARY KEY (`id`),
    UNIQUE KEY `uk_sys_user_role` (`user_id`, `role_id`),
    KEY `idx_sys_user_role_user_id` (`user_id`),
    KEY `idx_sys_user_role_role_id` (`role_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_general_ci COMMENT='用户角色关联表';

-- =====================================================
-- 4. 部门表 (sys_dept)
-- 用途: 企业组织架构树，支持无限级父子层级
-- =====================================================
CREATE TABLE `sys_dept` (
    `id` BIGINT UNSIGNED NOT NULL AUTO_INCREMENT COMMENT '主键ID',
    `parent_id` BIGINT UNSIGNED NOT NULL DEFAULT 0 COMMENT '上级部门ID: 0表示一级部门 (FK → sys_dept.id)',
    `dept_name` VARCHAR(50) NOT NULL COMMENT '部门名称（同级不可重名）',
    `dept_code` VARCHAR(50) NOT NULL COMMENT '部门编码（全局唯一）: DEPT_XXXX',
    `leader_id` BIGINT UNSIGNED DEFAULT NULL COMMENT '部门负责人ID (FK → sys_user.id)',
    `sort_order` INT UNSIGNED NOT NULL DEFAULT 0 COMMENT '同级排序号: 0-9999，值越小越靠前',
    `status` TINYINT UNSIGNED NOT NULL DEFAULT 1 COMMENT '部门状态: 0-禁用(不可新领用), 1-启用',
```

### 5.3 容错设计

| 场景 | 策略 | 说明 |
|------|------|------|
| 缓存穿透 | 空值缓存（1min过期） | 查询不存在的数据时缓存空值 `"null"` |
| 缓存击穿 | 分布式锁控制回源 | 热点Key过期时仅一个线程查DB，其余等待 |
| 缓存雪崩 | 过期时间±10%随机偏移 | 300s → 270~330s，防集中过期 |
| Redis不可用 | 自动降级查MySQL | 捕获连接异常后直接查DB，记录ERROR日志 |
| Redis恢复 | 30秒探测一次 | 恢复后自动回切启用缓存 |

### 5.4 分布式锁实现

基于 Lettuce `SET key value NX EX seconds` 原子命令实现：

```java
public class RedisUtil {
    /**
     * 获取分布式锁
     * @param key 锁Key
     * @param waitSec 等待时间(秒)，获取不到时最多等N秒
     * @param holdSec 持有时间(秒)，超时自动释放防死锁
     * @return lockValue(UUID)，用于释放时校验归属；获取失败返回null
     */
    public static String tryLock(String key, int waitSec, int holdSec) {
        String lockValue = UUID.randomUUID().toString();
        long deadline = System.currentTimeMillis() + waitSec * 1000L;
        while (System.currentTimeMillis() < deadline) {
            String result = redisTemplate.opsForValue()
                .setIfAbsent(key, lockValue, Duration.ofSeconds(holdSec));
            if ("OK".equals(result)) {
                return lockValue;
            }
            Thread.sleep(50); // 50ms重试间隔
        }
        return null;
    }

    /**
     * 释放分布式锁（Lua脚本原子操作，校验归属）
     */
    public static void unlock(String key, String lockValue) {
        String script = "if redis.call('get', KEYS[1]) == ARGV[1] " +
                        "then return redis.call('del', KEYS[1]) " +
                        "else return 0 end";
        redisTemplate.execute(
            new DefaultRedisScript<>(script, Long.class),
            Collections.singletonList(key), lockValue);
    }
}
```

### 6.1 登录认证模块

**核心流程**：

```

  │
  ├─ 1. 校验参数非空
```

```java
@Component
public class JwtAuthenticationFilter extends OncePerRequestFilter {
    @Override
    protected void doFilterInternal(HttpServletRequest request,
            HttpServletResponse response, FilterChain chain) {
        String token = extractToken(request);
        if (token == null) {
            write401(response, "未登录");
            return;
        }
        // 1. 校验Token黑名单
        String jti = JwtUtil.getJti(token);
        if (RedisUtil.exists("eams:token:blacklist:" + jti)) {
            write401(response, "登录已过期，请重新登录");
            return;
        }
        // 2. 校验Token签名与有效期
        if (!JwtUtil.verify(token)) {
            write401(response, "登录已过期，请重新登录");
            return;
        }
        // 3. 设置SecurityContext
        SecurityContextHolder.set(JwtUtil.parse(token));
        chain.doFilter(request, response);
    }
}
```

```java
public class RoleAspect {

    public Object checkRole(ProceedingJoinPoint joinPoint, RequireRole requireRole) {
        Set<String> userRoles = SecurityContextHolder.getCurrentRoles();
        boolean hasRole = Arrays.stream(requireRole.value())
            .anyMatch(userRoles::contains);
```

```java
        String yyMM = LocalDate.now().format(DateTimeFormatter.ofPattern("yyMM"));
        String lockKey = "eams:lock:asset:code:" + categoryShort + ":" + yyMM;
        String lockValue = RedisUtil.tryLock(lockKey, 3, 30);
        if (lockValue == null) {
            throw new BusinessException(400, "编码生成繁忙，请稍后重试");
        }
        try {
            // 查询当日该类别最大流水号
            String maxCode = assetMapper.selectMaxCodeByPrefix(
                "AS-" + categoryShort + "-" + yyMM);
            int seq = 1;
            if (maxCode != null) {
                seq = Integer.parseInt(maxCode.substring(maxCode.length() - 4)) + 1;
            }
            return String.format("AS-%s-%s-%04d", categoryShort, yyMM, seq);
        } finally {
            RedisUtil.unlock(lockKey, lockValue);
        }
    }
}
```

### 6.4 折旧计提定时任务

```java
@Component
public class DepreciationScheduledTask {
    @Scheduled(cron = "0 0 2 1 * ?") // 每月1日凌晨2:00执行
    @Transactional
    public void executeMonthlyDepreciation() {
        // 1. 查询需计提折旧的资产（状态非报废/非盘点中，折旧未满）
        List<AssetInfo> assets = assetMapper.selectForDepreciation();
        // 2. 逐资产计算并插入折旧记录
        for (AssetInfo asset : assets) {
            BigDecimal monthlyAmount = calculateMonthlyAmount(asset);
            int executedMonths = assetDepreciationMapper.countByAssetId(asset.getId());
            BigDecimal accumulated = monthlyAmount.multiply(
                new BigDecimal(executedMonths + 1));
            BigDecimal netValue = asset.getOriginalValue().subtract(accumulated);
            // 净值<=残值时停止
            BigDecimal residualValue = asset.getOriginalValue()
                .multiply(asset.getResidualRate().divide(new BigDecimal("100")));
            if (netValue.compareTo(residualValue) <= 0) continue;
            AssetDepreciation record = new AssetDepreciation();
            record.setAssetId(asset.getId());
            record.setDepreciationMonth(LocalDate.now().format(
                DateTimeFormatter.ofPattern("yyyy-MM")));
            record.setMonthlyAmount(monthlyAmount);
            record.setAccumulated(accumulated);
            record.setNetValue(netValue);
            record.setStatus(1);
            assetDepreciationMapper.insert(record);
        }
    }
}
```

**DeepSeek API调用工具类**：

```java
public class AIClientUtil {
    private static final String API_URL = "https://api.deepseek.com/v1/chat/completions";
    private static final String API_KEY = "${deepseek.api.key}";
    private static final int TIMEOUT_SECONDS = 15;
    private static final int MAX_RETRIES = 3;

    public static String call(String systemPrompt, String userQuery) {
        for (int attempt = 1; attempt <= MAX_RETRIES; attempt++) {
            try {
                HttpResponse response = HttpUtil.createPost(API_URL)
                    .header("Authorization", "Bearer " + API_KEY)
                    .header("Content-Type", "application/json")
                    .body(buildRequestBody(systemPrompt, userQuery))
                    .timeout(TIMEOUT_SECONDS * 1000)
                    .execute();
                if (response.isOk()) {
                    return extractContent(response.body());
                }
            } catch (Exception e) {
                if (attempt == MAX_RETRIES) {
                    // 连续3次失败 → 触发降级
                    RedisUtil.set("eams:ai:degraded", "1", 300);
                    throw new BusinessException(503,
                        "AI智能查询服务暂时不可用，请使用传统筛选查询");
                }
            }
        }
        return null;
    }
}
```

```
正常状态          →  连续3次调用失败    →  降级状态
(正常服务)          (或日调用超5000次)     (返回503提示)
      ↑                                        │
      │          每5分钟自动探测                │
      └────────────────────────────────────────┘
           探测成功 → 清除 eams:ai:degraded → 恢复正常
           探测失败 → 续期 eams:ai:degraded 5分钟 → 保持降级
```

## 8. 安全方案

### 8.1 认证安全

| 措施 | 实现 |
|------|------|
| 密码存储 | BCrypt加密，工作因子 cost=10 |
| 密码复杂度 | 6-20位，含字母和数字，前后端双重校验 |
| JWT签名 | HMAC-SHA256，密钥存储在配置中心（不可硬编码） |
| Token有效期 | 普通2小时，记住我7天 |
| Token刷新 | 一期不支持Token刷新，过期后重新登录 |
| 主动登出 | JTI加入Redis黑名单，有效期=Token剩余时间 |
| 登录锁定 | 5次失败锁30分钟，Redis计数器 |

### 8.2 接口安全

| 措施 | 实现 |
|------|------|
| 接口鉴权 | JwtAuthFilter 校验所有业务接口（除 `/api/login`） |
| RBAC | `@RequireRole` 注解 + AOP 切面 |
| 数据权限 | 部门管理员/普通员工在Service层自动拼接数据过滤条件 |
| SQL注入 | MyBatis-Plus `#{}` 参数化查询，禁止 `${}` 拼接 |
| XSS | 前端 input 转义 + 后端 `HtmlUtils.htmlEscape()` |
| CSRF | SPA + JWT Bearer Token，天然免疫（不使用Cookie传递Token） |
| 文件上传 | 白名单校验扩展名 + 魔数校验文件真实类型 + 大小限制 |
| CORS | 仅允许前端域名，`Access-Control-Allow-Origin` 白名单 |

### 8.3 数据安全

| 措施 | 实现 |
|------|------|
| 脱敏 | 操作日志中手机号 `138****5678` |
| 审计 | 所有增删改操作写入 `sys_operation_log`，含变更前后数据 |
| 登录记录 | 每次登录写入 `sys_login_log`（含IP、UA） |
| 敏感配置 | 数据库密码、Redis密码、DeepSeek API Key 通过环境变量注入 |

## 11. 风险预案

| 序号 | 风险项 | 影响 | 概率 | 应对措施 |
|:----:|--------|------|:----:|----------|
| 1 | Redis宕机 | 缓存不可用、锁失效 | 低 | 自动降级查MySQL，锁用数据库行锁兜底；Redis哨兵/Cluster（二期） |
| 2 | MySQL性能瓶颈 | 查询变慢 | 中 | 慢SQL监控、索引优化、读写分离（二期） |
| 3 | DeepSeek API不可用 | AI查询不可用 | 中 | 自动降级：连续3次失败后禁用AI入口，5分钟自动探测恢复 |
| 4 | 文件上传磁盘满 | 上传失败 | 低 | 监控磁盘使用率 >80%告警、定时清理临时文件 |
| 5 | 并发领用冲突 | 同一资产被多人同时领用 | 低 | Redis分布式锁 + 数据库唯一约束双重防护 |
| 6 | 编码重复 | 并发生成资产编码冲突 | 低 | Redis分布式锁保护编码流水号 |
| 7 | 盘点期间资产操作 | 盘点时资产被领用/编辑 | 低 | 盘点任务创建后资产批量锁定(状态=盘点中) |
| 8 | Token泄露 | 未授权访问 | 低 | Token短有效期(2h)、主动登出黑名单、IP异常检测记录 |
| 9 | 日志表过大 | 查询性能下降 | 中 | 日志表按月分表或定时归档（二期）；一期设12个月保留告警 |
| 10 | 数据误删 | 业务数据丢失 | 低 | 全量软删除，可手动恢复 `is_deleted=0`；每日数据库全量备份 |
