## 2. scrap_order 数据表结构

### 2.1 完整建表SQL

见项目 `sql/init.sql` 第529-558行，与技术方案4.3.8完全一致

### 2.2 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `id` | BIGINT UNSIGNED | ✅ | 主键自增 |
| `scrap_no` | VARCHAR(30) | ✅ | 报废编号: `BF-YYYYMMDD-XXXX` |
| `asset_id` | BIGINT UNSIGNED | ✅ | 报废资产ID (FK → asset_info.id) |
| `scrap_reason` | VARCHAR(20) | ✅ | 报废原因: 老化损坏/技术淘汰/维修成本过高/盘亏确认/其他 |
| `reason_desc` | VARCHAR(500) | ✅ | 原因详细说明: 10-500字符 |
| `disposal_advice` | VARCHAR(20) | ✅ | 处置建议: 变卖/回收/销毁/其他 |
| `status` | TINYINT | ✅ | 报废状态: 0-待初审, 1-待终审, 2-已通过(待处置), 3-已驳回, 4-已处置 |
| `applicant_id` | BIGINT UNSIGNED | ✅ | 申请人ID (FK → sys_user.id) |
| `remark` | VARCHAR(200) | 否 | 申请备注（驳回时用于存储驳回原因，PRD 6.9.2） |
| `disposal_method` | VARCHAR(20) | 否 | 实际处置方式 |
| `disposal_date` | DATE | 否 | 处置日期 |
| `disposal_income` | DECIMAL(12,2) | 否 | 处置收入(元) |
| `disposal_cost` | DECIMAL(12,2) | 否 | 处置费用(元) |
| `disposal_handler` | VARCHAR(50) | 否 | 经办人姓名 |
| `disposal_desc` | VARCHAR(500) | 否 | 处置过程说明 |
| `attachment_urls` | VARCHAR(1000) | 否 | 附件URL（逗号分隔） |
| `version` | INT | ✅ | 乐观锁版本号 |
| `create_time` | DATETIME | ✅ | 创建时间 |
| `create_by` | VARCHAR(50) | 否 | 创建人 |
| `update_time` | DATETIME | ✅ | 更新时间 |
| `update_by` | VARCHAR(50) | 否 | 更新人 |
| `is_deleted` | TINYINT | ✅ | 软删除: 0-正常, 1-已删除 |

### 2.3 与 asset_info 关联关系

```
scrap_order.asset_id → asset_info.id
  ├─ 报废申请时: 校验资产状态（不可为4-报废/5-盘点中）
  ├─ 初审通过后: 资产状态不变
  ├─ 终审通过后: 资产状态更新为 4-报废，停止折旧计提
  └─ 处置完成后: 资产归档，仅可查询
```

## 3. 全部后端接口清单

### 3.1 报废申请

| 项目 | 内容 |
|------|------|
| **接口** | `POST /api/scrap/apply` |
| **权限** | `ROLE_SUPER_ADMIN` / `ROLE_ASSET_ADMIN` / `ROLE_DEPT_ADMIN` |
| **说明** | 提交报废申请，生成 BF-YYYYMMDD-XXXX 编号 |
| **状态流转** | → status=0 (待初审) |

**请求体 (JSON)**:
```json
{
  "assetId": 1001,
  "scrapReason": "老化损坏",
  "reasonDesc": "该设备已使用5年，硬盘出现坏道，无法修复",
  "disposalAdvice": "回收",
  "attachmentUrls": "/uploads/2026/06/img001.jpg,/uploads/2026/06/doc001.pdf",
  "remark": "建议尽快处理"
}
```

**返回示例**:
```json
{
  "code": 200,
  "message": "报废申请已提交，请等待审批",
  "data": "BF-20260629-0001"
}
```

**错误响应**:
| code | message | 触发条件 |
|:----:|---------|----------|
| 400 | "该资产已报废，不可重复申请" | 资产状态=4 |
| 400 | "该资产正在盘点中，不可报废" | 资产状态=5 |
| 400 | "该资产存在待审批的报废单，请勿重复提交" | 已有 status=0/1 的报废单 |
| 400 | "操作繁忙，请稍后重试" | Redis锁获取失败 |
| 403 | "您没有权限执行此操作" | 部门管理员非本部门资产 |

### 3.3 报废审批操作

| 项目 | 内容 |
|------|------|
| **接口** | `PUT /api/scrap/approve` |
| **权限** | `ROLE_ASSET_ADMIN` (初审) / `ROLE_SUPER_ADMIN` (终审) |
| **请求体** | `{ scrapId, approvalResult, rejectReason?, approveOpinion? }` |

**审批通过 (approvalResult=1)**:
```json
// 请求
{ "scrapId": 1, "approvalResult": 1, "approveOpinion": "情况属实，同意报废" }

// 初审通过 → status=1: "初审通过，待超级管理员终审"
// 终审通过 → status=2: "报废审批通过，资产已标记为报废，请执行处置"
```

**审批驳回 (approvalResult=0)**:
```json
// 请求
{ "scrapId": 1, "approvalResult": 0, "rejectReason": "该设备仍有使用价值，建议继续使用或调拨至其他部门" }

// 返回
{ "code": 200, "message": "已驳回该报废申请" }
```

**错误响应**:
| code | message | 触发条件 |
|:----:|---------|----------|
| 400 | "驳回原因为10-200个字符" | 驳回原因不满足长度要求 |
| 400 | "当前报废单状态不可审批" | 状态不是0或1 |
| 400 | "不可审批自己提交的报废申请" | 申请人=审批人 |
| 403 | "仅资产管理员可进行初审" | 角色不符 |
| 403 | "仅超级管理员可进行终审" | 角色不符 |

## 4. Redis 分布式锁说明

### 4.1 锁配置

| 配置项 | 值 | 说明 |
|--------|-----|------|
| Key规则 | `eams:lock:scrap:{assetId}` | 同一资产加锁 |
| 等待时间 | 3秒 | `LOCK_WAIT_SEC` |
| 持有时间 | 30秒 | `LOCK_HOLD_SEC` |
| 释放方式 | Lua脚本原子校验+删除 | `RedisUtil.unlock(key, lockValue)` |

### 4.2 使用场景

1. **报废申请** (`ScrapServiceImpl.apply()`): 同一资产不可同时发起多条待审批报废单据
2. **锁获取失败**: 返回 `"操作繁忙，请稍后重试"` (code=400)

### 4.3 锁实现代码位置

- `com.example.eams.common.util.RedisUtil.tryLock()`
- `com.example.eams.common.util.RedisUtil.unlock()`
- 使用位置: `com.example.eams.scrap.service.impl.ScrapServiceImpl.apply()`

## 5. 后端接口自测清单

### 5.1 正常完整流程

| 步骤 | 接口 | 操作 | 预期结果 | 验证点 |
|:----:|------|------|----------|--------|
| 1 | POST /api/scrap/apply | 提交报废申请 | 返回 scrapNo | status=0, 资产状态不变 |
| 2 | GET /api/scrap/approval/list | 资产管理员查看 | 列表包含该申请 | status=0 待初审 |
| 3 | PUT /api/scrap/approve | 资产管理员初审通过 | approvalResult=1 | status→1 待终审 |
| 4 | GET /api/scrap/approval/list | 超级管理员查看 | 列表包含该申请 | status=1 待终审 |

| 6 | GET /api/scrap/disposal/list | 查看待处置列表 | 列表包含该记录 | status=2 |
| 7 | PUT /api/scrap/disposal | 处置登记 | 提交处置信息 | status→4 已处置 |
| 8 | GET /api/scrap/record/list | 查看报废记录 | 列表包含完整记录 | status=4 |

### 5.2 重复提交报废锁拦截

| 步骤 | 操作 | 预期 |
|:----:|------|------|

## 8. 联调常见报错及排查方案

### 8.1 报废申请失败

| 错误信息 | 原因 | 排查方案 |
|----------|------|----------|
| "该资产已报废" | 资产 status=4 | 确认资产状态，选择其他资产 |
| "该资产正在盘点中" | 资产 status=5 | 等待盘点结束后操作 |
| "请勿重复提交" | 已有待审批报废单 | 取消原报废单或等待审批 |
| "操作繁忙" | Redis锁冲突 | 稍后重试，检查Redis连接 |

### 8.2 审批失败

| 错误信息 | 原因 | 排查方案 |
|----------|------|----------|
| "驳回原因为10-200" | 驳回原因长度不足 | 填写≥10字符的驳回原因 |
| "当前报废单状态不可审批" | 状态已被他人修改 | 刷新列表重新操作 |
| "不可审批自己的申请" | 申请人=审批人 | 换其他管理员审批 |
| "仅资产管理员可进行初审" | 角色不符 | 检查用户角色配置 |
| "仅超级管理员可进行终审" | 角色不符 | 使用admin账号操作 |

### 8.3 处置登记失败

| 错误信息 | 原因 | 排查方案 |
|----------|------|----------|
| "仅已通过的报废单" | 报废单状态≠2 | 确认报废单已完成终审 |
| "处置日期不能晚于当前" | 选择了未来日期 | 选择今天或之前的日期 |
| "处置收入不能为负数" | 输入了负值 | 填写≥0的金额 |

### 8.4 前端显示异常

| 现象 | 可能原因 | 排查方案 |
|------|----------|----------|
| 菜单不显示 | 角色无权限 | 检查 sys_menu 权限配置 |
| 页面403 | 路由守卫拦截 | 检查 meta.perm 配置 |
| 列表为空 | 筛选条件过严 | 清空筛选条件重试 |
| 状态标签颜色不对 | StatusTag未注册scrap类型 | 确认StatusTag.vue已添加SCRAP_MAP |
| 附件下载失败 | URL格式不对 | 检查urls路径/文件名 |

### 8.5 Redis 相关

| 现象 | 原因 | 排查方案 |
|------|----------|----------|
| 锁获取失败 | Redis不可用或并发冲突 | 检查Redis连接，降低并发 |
| 锁未释放 | 业务逻辑未执行finally | 检查代码逻辑 |
| 误删他人锁 | 未校验lockValue | 确认unlock使用Lua脚本 |
