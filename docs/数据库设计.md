## 2. 设计规范

### 2.1 通用审计字段

所有业务表统一包含：

| 字段 | 类型 | 说明 |
|------|------|------|
| `create_time` | DATETIME | 创建时间，DEFAULT CURRENT_TIMESTAMP |
| `create_by` | VARCHAR(50) | 创建人用户名 |
| `update_time` | DATETIME | 修改时间，ON UPDATE CURRENT_TIMESTAMP |
| `update_by` | VARCHAR(50) | 修改人用户名 |
| `is_deleted` | TINYINT UNSIGNED | 软删除标记：0-正常，1-已删除，DEFAULT 0 |

### 2.2 命名规范

| 规范项 | 规则 | 示例 |
|--------|------|------|
| 表名 | `{前缀}_{业务名}` | `sys_user`, `asset_info` |
| 主键 | `id` BIGINT UNSIGNED AUTO_INCREMENT | — |
| 逻辑外键 | `{关联表名}_id`，COMMENT标注 `FK → {目标表}` | `dept_id` |
| 索引 | `idx_{表名}_{字段}` / `uk_{表名}_{字段}` | `idx_asset_info_status` |
| 业务编号 | `*_no` / `*_code` 独立字段 | `asset_code`, `apply_no` |

### 2.3 表前缀

| 前缀 | 类别 | 表数 | 示例 |
|------|------|:----:|------|
| `sys_` | 系统权限 + 系统日志 | 12 | `sys_user`, `sys_login_log` |
| `asset_` | 资产台账 | 3 | `asset_info` |
| `proc_` | 采购入库 | 2 | `proc_order`, `proc_supplier` |
| `req_` | 领用归还 | 2 | `req_order`, `req_approval_log` |
| `trans_` | 资产调拨 | 1 | `trans_order` |
| `repair_` | 维保报修 | 2 | `repair_order`, `repair_record` |
| `scrap_` | 报废处置 | 1 | `scrap_order` |
| `inv_` | 盘点管理 | 3 | `inv_task`, `inv_detail`, `inv_difference` |
| `ai_` | AI查询 | 1 | `ai_query_log` |

## 4. 表清单

### 4.1 系统权限表（10张）

| 表名 | 说明 | 预置数据 | 关键索引 |
|------|------|:--------:|----------|
| `sys_user` | 用户表 | 5条 | uk:username; idx:dept_id,phone,status,real_name |
| `sys_role` | 角色表 | 4条 | uk:role_code |
| `sys_user_role` | 用户-角色关联 | 5条 | uk:(user_id,role_id); idx:user_id,role_id |
| `sys_dept` | 部门表 | 8条 | uk:dept_code; idx:parent_id,status |
| `sys_menu` | 菜单权限表 | 87条 | uk:(parent_id,menu_name); idx:parent_id,menu_type |
| `sys_role_menu` | 角色-菜单关联 | 约250条 | uk:(role_id,menu_id) |
| `sys_dict_type` | 字典类型表 | 4条 | uk:dict_code |
| `sys_dict_item` | 字典项表 | 16条 | uk:(dict_code,dict_value); idx:dict_code,sort_order |
| `sys_config` | 系统参数表 | 21条 | uk:param_key; idx:param_group |
| `sys_config_history` | 参数变更历史 | 0条 | idx:config_id,param_key,create_time |

### 4.2 系统日志表（2张）

| 表名 | 说明 | 关键索引 |
|------|------|----------|
| `sys_operation_log` | 操作日志（不可删除） | idx:operator,module,action_type,create_time |
| `sys_login_log` | 登录日志 | idx:username,login_status,create_time |

### 4.3 资产台账表（3张）

| 表名 | 说明 | 关键索引 | 特殊字段 |
|------|------|----------|----------|
| `asset_info` | 资产信息（核心） | uk:asset_code; idx:category,status,dept_id,user_id,purchase_date,location,asset_name; idx:(status,dept_id) | `version`乐观锁 |
| `asset_attachment` | 资产附件 | idx:asset_id,file_type | — |
| `asset_depreciation` | 折旧明细 | uk:(asset_id,depreciation_month); idx:asset_id,month | — |

### 4.4 业务单据表（12张）

| 表名 | 模块 | 状态流转 | 特殊字段 |
|------|------|----------|----------|
| `proc_order` | 采购入库 | 待验收→已验收→已入库/已取消 | `quantity`批量入库 |
| `proc_supplier` | 供应商 | 启用/禁用 | — |
| `req_order` | 领用归还 | 待部门审批→待资产管理员审批→已通过→已归还/已驳回 | `version`乐观锁 |
| `req_approval_log` | 领用审批日志 | 通过/驳回 | — |
| `trans_order` | 资产调拨 | 待调入确认→待资产管理员审批→已通过/已驳回 | `version`乐观锁 |
| `repair_order` | 维保报修 | 待维修→维修中→已修复/无法修复 | `pre_repair_status`恢复 |
| `repair_record` | 维修记录 | — | — |
| `scrap_order` | 报废处置 | 待初审→待终审→已通过→已处置/已驳回 | `version`乐观锁 |
| `inv_task` | 盘点任务 | 进行中→已完成/已取消 | `total_count`等统计字段 |
| `inv_detail` | 盘点明细 | 未确认→已确认 | 正常/盘盈/盘亏 |
| `inv_difference` | 盘点差异 | 待处理→已处理 | — |
| `ai_query_log` | AI查询日志 | SUCCESS/FAIL/TIMEOUT/RATE_LIMITED | — |

## 5. 核心表字段说明

### 5.1 sys_user（用户表）

| 字段 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `username` | VARCHAR(50) | ✅ | 登录账号，全局唯一 |
| `password` | VARCHAR(200) | ✅ | BCrypt加密 |
| `real_name` | VARCHAR(50) | ✅ | 员工真实姓名 |
| `dept_id` | BIGINT UNSIGNED | ❌ | FK → sys_dept.id |
| `phone` | VARCHAR(20) | ❌ | 手机号，全局唯一校验 |
| `email` | VARCHAR(100) | ❌ | 邮箱，全局唯一校验 |
| `status` | TINYINT UNSIGNED | ✅ | 0-禁用, 1-启用（默认1） |
| `last_login_time` | DATETIME | ❌ | 最近登录时间 |
| `last_login_ip` | VARCHAR(50) | ❌ | 最近登录IP |

### 5.2 asset_info（资产信息表）

| 字段 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `asset_code` | VARCHAR(30) | ✅ | 全局唯一编码 AS-IT-2606-0001 |
| `asset_name` | VARCHAR(100) | ✅ | 资产名称 |
| `category` | VARCHAR(50) | ✅ | 分类（字典值） |
| `original_value` | DECIMAL(12,2) | ✅ | 采购原值（元） |
| `purchase_date` | DATE | ✅ | 采购日期，折旧从次月计提 |
| `useful_life` | TINYINT UNSIGNED | ✅ | 使用年限 1-50 |
| `residual_rate` | DECIMAL(5,2) | ✅ | 净残值率 默认5% |
| `dept_id` | BIGINT UNSIGNED | ✅ | FK → sys_dept.id |
| `user_id` | BIGINT UNSIGNED | ❌ | FK → sys_user.id，闲置为NULL |
| `status` | TINYINT UNSIGNED | ✅ | 0-闲置,1-在用,2-借用,3-维修,4-报废,5-盘点中 |
| `version` | INT UNSIGNED | ✅ | 乐观锁，每次UPDATE+1 |

### 5.3 req_order（领用申请单表）

| 字段 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `apply_no` | VARCHAR(30) | ✅ | RY-YYYYMMDD-XXXX |
| `asset_id` | BIGINT UNSIGNED | ✅ | FK → asset_info.id |
| `applicant_id` | BIGINT UNSIGNED | ✅ | FK → sys_user.id |
| `status` | TINYINT UNSIGNED | ✅ | 0-待部门审批,1-待资产管理员审批,2-已通过,3-已驳回,4-已归还 |
| `expect_return_date` | DATE | ✅ | 预计归还日期 |
| `return_date` | DATE | ❌ | 实际归还日期 |
| `return_asset_status` | TINYINT UNSIGNED | ❌ | 0-完好,1-有损坏 |
| `version` | INT UNSIGNED | ✅ | 乐观锁 |

## 8. 使用说明

### 8.1 初始化数据库

```bash
mysql -u root -p < sql/init.sql
```

### 8.2 默认密码

所有演示账号初始密码均为 `Eams@123456`。

> ⚠️ init.sql 中的BCrypt密码哈希为占位值。生产部署前请执行以下代码生成真实哈希替换：

```java
String hash = BCrypt.hashpw("Eams@123456", BCrypt.gensalt(10));
System.out.println(hash);
```

### 8.3 验证

```sql
-- 验证表数量 (应为27)
SELECT COUNT(*) FROM information_schema.tables

UNION ALL SELECT '字典类型', COUNT(*) FROM sys_dict_type
UNION ALL SELECT '字典项', COUNT(*) FROM sys_dict_item
UNION ALL SELECT '系统参数', COUNT(*) FROM sys_config
```
