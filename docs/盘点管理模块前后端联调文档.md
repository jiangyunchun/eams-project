## 1. 盘点全业务流程

```
┌────────────────────────────────────────────────────────────────────┐
│                      盘点管理全生命周期                               │
├────────────────────────────────────────────────────────────────────┤
│                                                                    │
│  ① 创建任务 ──→ ② 锁定资产(盘点中) ──→ ③ 逐项核对执行盘点           │
│       │                                      │                    │
│       │  Redis锁防重创建                      ├ 正常 → 确认        │
│       │  批量更新资产status=5                 ├ 盘盈 → 登记入库     │
│       │  生成明细快照                         └ 盘亏 → 登记原因     │
│       │                                      │                    │
│       ▼                                      ▼                    │
│  [取消盘点]                              ④ 完成盘点               │
│       │                                      │                    │
│       │  恢复资产原状态                        ├ 汇总数据           │
│       │  删除盘盈入库资产                      ├ 盘盈生成资产台账    │
│       │                                      ├ 盘亏生成差异记录    │
│       ▼                                      ├ 正常资产恢复原状态  │
│  任务→已取消                                │                    │
│                                              ▼                    │
│                                       ⑤ 差异处理                  │
│                                              │                    │
│                                              ├ 查看差异列表       │
│                                              ├ 逐条/批量标记已处理 │
│                                              └ Excel导出          │
│                                                                    │
│  闭环: 创建 → 执行 → 盘盈入库/盘亏记录 → 差异处理 → 归档           │
└────────────────────────────────────────────────────────────────────┘
```

## 4. 接口入参/返回JSON示例

### 4.1 创建盘点任务

**POST** `/api/inventory/task/add`

```json
// 请求
{
  "taskName": "2026年Q3全公司资产盘点",
  "scopeType": "DEPT",
  "deptIds": [1, 2],
  "inventoryDate": "2026-09-30",
  "remark": "季度例行盘点"
}

// 响应
{
  "code": 200,
  "message": "盘点任务【2026年Q3全公司资产盘点】创建成功，范围内共45项资产",
  "data": {
    "id": 1,
    "taskNo": "PD-20260630-001",
    "taskName": "2026年Q3全公司资产盘点",
    "scopeType": "DEPT",
    "scopeValue": "{\"deptIds\":[1,2]}",
    "inventoryDate": "2026-09-30",
    "totalCount": 45,
    "checkedCount": 0,
    "status": 0
  }
}
```

### 4.2 确认盘点明细

**PUT** `/api/inventory/execute/confirm`

```json
// 请求(正常)
{
  "taskId": 1,
  "detailIds": [1, 2, 3],
  "inventoryResult": 2,
  "remark": ""
}

// 请求(盘亏)
{
  "taskId": 1,
  "detailIds": [5],
  "inventoryResult": 1,
  "remark": "实物未找到，可能已被借用未登记"
}

// 响应
{
  "code": 200,
  "message": "已确认1项资产，结果: 盘亏",
  "data": null
}
```

## 5. Redis分布式锁

| 锁Key | 等待 | 持有 | 使用场景 |
|-------|:----:|:----:|----------|
| `eams:lock:inventory:create:{scopeType}:{scopeValue}` | 3s | 30s | 创建盘点任务时，防同范围重复创建 |
| `eams:lock:inventory:{taskId}` | 3s | 30s | 执行盘点/确认明细/完成盘点/盘盈登记 |
| `eams:lock:asset:code:{categoryCode}:{yyMM}` | 3s | 30s | 盘盈资产编码生成，防编码重复 |

**缓存清除**: 创建/取消/完成盘点任务后，执行 `RedisUtil.deleteByPattern("eams:asset:list:*")` 批量清除资产列表缓存。

## 9. 常见联调报错及解决方案

| # | 错误现象 | 可能原因 | 解决方案 |
|:--:|----------|----------|----------|
| 1 | 创建任务返回 "该范围内已存在进行中的盘点任务" | 数据库已有同范围进行中任务 | 取消旧任务或等待完成后再创建 |
| 2 | 创建任务返回 "操作繁忙，请稍后重试" | Redis锁竞争 | 等待3秒后重试 |
| 3 | 盘盈登记返回 "编码生成繁忙" | 资产编码锁被占用 | 等待3秒后重试 |
| 4 | 完成盘点返回 "还有X项资产未确认" | 有未确认的明细 | 先确认全部明细再完成 |
| 5 | 获取明细列表为空 | 任务已取消或ID错误 | 检查taskId是否正确 |
| 6 | 资产状态一直为"盘点中" | 取消/完成操作未执行 | 确认完成或取消流程 |
| 7 | 差异列表无数据 | 盘点无盘盈/盘亏 | 正常，无差异则无记录 |
| 8 | 前端表格不显示 | 接口返回字段名与前端不匹配 | 检查前后端字段映射表 |
| 9 | 状态标签颜色不对 | StatusTag type参数错误 | 确认使用 `inventory-task` 或 `inventory-result` |
| 10 | 401未授权 | Token过期或未登录 | 重新登录 |
| 11 | 403无权限 | 角色权限不足 | 使用admin/assetadmin账号 |
| 12 | 部门管理员查不到数据 | scopeValue中不含该部门ID | 检查任务创建时的部门范围 |
| 13 | 盘亏资产仍显示为盘点中 | 资产未在完成时处理 | 确认完成了盘点任务 |
