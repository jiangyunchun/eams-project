# EAMS 系统管理模块 前后端联调核对清单

---

## 一、登录认证

| 检查项 | 前端 | 后端接口 | 易出错点 |
|--------|------|----------|----------|
| 登录请求 | `POST /api/login` body: {username, password, rememberMe} | `LoginController.login()` | `rememberMe` 为 true 时 Token 有效期 7 天，否则 2 小时 |
| 登录成功响应 | 返回 {token, userInfo{id,username,realName,deptId,roles,permissions}} | `LoginService.login()` | `roles` 是 Set<String> 角色编码集合 |
| 登录失败 | 统一弹 "用户名或密码错误" | 返回 400，不区分用户不存在/密码错误 | 后端不需要区分，message 固定 |
| 连续5次错误 | Redis 锁定30分钟 | `eams:login:lock:{username}` | 提示含剩余分钟数 |
| 禁用账号 | 弹 "账号已被禁用" | 返回 403 | / |
| Token 黑名单 | 登出后 JTI 加入 Redis，有效期=Token剩余时间 | `/api/logout` | 与前端 Token 过期时间保持一致 |
| 401 拦截 | Axios 拦截器跳转 `/login` | `JwtAuthenticationFilter` | 白名单路径不需要 Token |

## 二、用户管理

| 检查项 | 前端 | 后端接口 | 易出错点 |
|--------|------|----------|----------|
| 列表查询 | `GET /api/system/user/list` params: {username, deptId, status, phone, pageNum, pageSize} | `UserController.list()` | `deptId` 传入 null 表示全部 |
| 新增用户 | `POST /api/system/user/add` | `UserController.add()` | 密码 BCrypt 加密，`roleIds` 为数组 |
| 编辑用户 | `PUT /api/system/user/edit` | `UserController.edit()` | 编辑时不传 password，不更新密码字段 |
| 删除用户 | `DELETE /api/system/user/delete/{id}` | `UserController.delete()` | 不可删除自己（前后端双重校验） |
| 重置密码 | `PUT /api/system/user/reset-pwd/{id}` | `UserController.resetPassword()` | 默认密码 "Eams@123456" |
| 状态切换 | `PUT /api/system/user/status` body: {id, status} | `UserController.toggleStatus()` | status: 0-禁用, 1-启用 |
| 用户详情 | `GET /api/system/user/detail/{id}` | `UserController.detail()` | 返回 {user, roleIds}，roleIds 用于回填多选 |
| 角色全量列表 | `GET /api/system/role/all` | `RoleController.all()` | 用于新增/编辑时的角色下拉框 |
| **校验规则** | | | |
| 用户名 | 4-20位字母开头 | `@Pattern("^[a-zA-Z][a-zA-Z0-9_]{3,19}$")` | 后端同样校验，唯一性 |
| 姓名 | 2-20字符 | `@Size(min=2, max=20)` | / |
| 密码 | 6-20位含字母和数字 | `@Pattern("^(?=.*[a-zA-Z])(?=.*\\d).{6,20}$")` | 仅新增时必填 |
| 手机号 | 11位 `1[3-9]` 开头 | `@Pattern("^1[3-9]\\d{9}$")` | 非必填，唯一性 |
| 邮箱 | 标准邮箱格式 | `@Email` | 非必填，唯一性 |
| 角色 | 至少一个 | `@NotEmpty` | 编辑时需重新提交完整 roleIds |
| **提示文案** | 新增成功 → "用户创建成功" | | |
| | 编辑成功 → "用户信息修改成功" | | |
| | 删除确认 → "确认删除用户【{姓名}】？删除后该账号将无法登录" | | |
| | 重置确认 → "确认重置用户【{姓名}】的密码？密码将重置为默认密码" | | |

## 三、角色管理

| 检查项 | 前端 | 后端接口 | 易出错点 |
|--------|------|----------|----------|
| 列表查询 | `GET /api/system/role/list` params: {roleName, roleCode, pageNum, pageSize} | `RoleController.list()` | / |
| 新增角色 | `POST /api/system/role/add` body: {roleName, roleCode, description} | `RoleController.add()` | roleCode 新增后不可修改 |
| 编辑角色 | `PUT /api/system/role/edit` body: {id, roleName, description} | `RoleController.edit()` | roleCode 不可修改 |
| 删除角色 | `DELETE /api/system/role/delete/{id}` | `RoleController.delete()` | 预置4角色不可删，有用户关联时拒绝 |
| 权限配置 | `PUT /api/system/role/permission` body: {roleId, menuIds} | `RoleController.assignPermission()` | menuIds 包含半选+全选节点 ID |
| 已有菜单ID | `GET /api/system/role/menu-ids/{roleId}` | `RoleController.getMenuIds()` | 返回 List<Long>，用于回显勾选 |
| 全量菜单树 | `GET /api/system/menu/tree` | `MenuController.tree()` | 数据含 M/P/B 三级，前端转树形 |
| **校验规则** | | | |
| 角色名称 | 2-20字符 | `@Size(min=2, max=20)` | / |
| 角色编码 | 大写字母+下划线 | `@Pattern("^[A-Z_]{1,50}$")` | 全局唯一，新增后不可改 |
| **提示文案** | 权限配置成功 → "权限配置保存成功" | | |

| 部门名称 | 2-20字符，同级不可重名 | 后端校验 | / |

## 通用注意事项

### 请求头
- 所有业务接口（除 `/api/login`、`/api/health`、`/api/system/dict/all`）需要 Header: `Authorization: Bearer {token}`

### 分页
- 请求参数: `pageNum=1&pageSize=10`
- 响应: `{ total, pageNum, pageSize, pages, list: [] }`

### 统一返回体
```json
{ "code": 200, "message": "操作成功", "data": {} }
```

### 操作确认弹窗文案（全部与 PRD 一致）
- 用户删除: "确认删除用户【{姓名}】？删除后该账号将无法登录"
- 用户禁用: "确认禁用用户【{姓名}】？禁用后该账号将无法登录"
- 角色删除: "确认删除角色【{名称}】？"
- 部门删除: "确认删除部门【{名称}】？删除后部门及关联数据将清除"
- 参数重置: "确认重置参数【{名称}】为默认值【{默认值}】？"
