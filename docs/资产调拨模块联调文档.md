# 资产调拨模块 — 前后端联调核对清单

> 模块版本：V1.0（Phase 1 二级审批） | 生成日期：2026-06-28

---

## 一、业务流程图

```
申请人登录 → [调拨申请] → 选择本部门闲置/在用资产 → 填写调入信息 → 提交
                                                              ↓
                                         trans_order.status = 0（待调入确认）
                                                              ↓
                    [调拨审批] ← 调入部门管理员确认 → status = 1（待资产管理员审批）
                                                              ↓
                    [调拨审批] ← 资产管理员终审 → status = 2（已通过）
                              ↓ 驳回                        ↓
                           status = 3                   asset_info 更新:
                                                         dept_id → to_dept_id
                                                         user_id → to_user_id
                                                         location → to_location
```

---

## 二、数据库表结构

### trans_order（调拨申请单）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT UNSIGNED | 主键（自增） |
| transfer_no | VARCHAR(30) | 调拨编号：`DB-YYYYMMDD-XXXX`（唯一） |
| asset_id | BIGINT UNSIGNED | FK → asset_info.id |
| from_dept_id | BIGINT UNSIGNED | 调出部门 → sys_dept.id |
| to_dept_id | BIGINT UNSIGNED | 调入部门 → sys_dept.id |
| to_user_id | BIGINT UNSIGNED | 调入使用人（可空） |
| to_location | VARCHAR(100) | 调入后存放地点 |
| transfer_reason | VARCHAR(500) | 调拨原因（10-500字符） |
| status | TINYINT | **0-待调入确认, 1-待资产管理员审批, 2-已通过, 3-已驳回** |
| remark | VARCHAR(200) | 备注 |
| applicant_id | BIGINT UNSIGNED | 申请人 → sys_user.id |
| version | INT | 乐观锁 |

### 接口详情

**1. 提交调拨申请**
```
POST /api/transfer/apply

Success:  200 "调拨申请已提交，请等待调入部门确认"
```

**4. 审批通过（两级）**
```
PUT /api/transfer/approval/pass
Request:  {"transferId": 1}
Level 1 (status=0→1): 200 "审批通过"  （确认调入）
Level 2 (status=1→2): 200 "审批通过"  （终审，资产划转更新asset_info三字段）
Error: 400 "该申请已被处理，无法重复审批"
```

## 七、Postman 测试用例

```bash
# 前置：登录
TOKEN=$(curl -s -X POST http://localhost:8080/api/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"Eams@123456"}' | grep -o '"token":"[^"]*"' | cut -d'"' -f4)

# 1. 提交调拨
curl -s -X POST http://localhost:8080/api/transfer/apply \
  -H "Authorization: Bearer $TOKEN" -H "Content-Type: application/json" \
  -d '{"assetId":59,"toDeptId":2,"toLocation":"Tech Room","transferReason":"Testing transfer module ten chars"}'
# → 200 "调拨申请已提交，请等待调入部门确认"

# 2. 一级审批（确认调入）
curl -s -X PUT http://localhost:8080/api/transfer/approval/pass \
  -H "Authorization: Bearer $TOKEN" -H "Content-Type: application/json" \
  -d '{"transferId":1}'
# → 200 "审批通过"

# 3. 二级审批（终审+资产划转）
curl -s -X PUT http://localhost:8080/api/transfer/approval/pass \
  -H "Authorization: Bearer $TOKEN" -H "Content-Type: application/json" \
  -d '{"transferId":1}'
# → 200 "审批通过"

# 4. 驳回（需填原因）
curl -s -X PUT http://localhost:8080/api/transfer/approval/reject \
  -H "Authorization: Bearer $TOKEN" -H "Content-Type: application/json" \
  -d '{"transferId":2,"rejectReason":"缺少必要的调入理由，请补充"}'
# → 200 "已驳回该调拨申请"

# 5. 查看详情
curl -s http://localhost:8080/api/transfer/record/detail/1 \
  -H "Authorization: Bearer $TOKEN"
```

## 八、联调常见坑点

| # | 问题 | 原因 | 处理 |
|---|------|------|------|
| 1 | 资产不可调拨 | status 不是 0 或 1 | 检查资产当前状态，维修/报废/盘点中不可调拨 |
| 2 | 调入部门=调出部门 | 校验拦截 | 前后端双重校验 |
| 3 | 重复提交 | 同资产有在途调拨单（status IN 0,1） | Redis 锁 + DB 查重双重防护 |
| 4 | 终审后资产未变更 | @Transactional 保证原子性 | 验证 asset_info.dept_id/location/user_id |
| 5 | StatusTag 显示不对 | 需要 `type="transfer"` | 已更新 StatusTag.vue 新增 TRANSFER_MAP |
