### 2.2 新增/编辑表单 `SupplierAddDTO` / `SupplierEditDTO`

| 前端字段 | 后端字段 | 类型 | 必填 | 校验规则 | 易错点 / 联调注意 |
|----------|---------|------|------|---------|-------------------|
| 供应商编码 | `supplierCode` | String | ✅ | 格式 `SUP-\d{4}`，全局唯一 | **编辑时前端禁用不可改**；唯一性校验含软删除数据（`is_deleted=0`） |
| 供应商名称 | `supplierName` | String | ✅ | 2-50 字符，全局唯一 | 唯一性校验排除自身 ID；前后端长度一致 |
| 联系人 | `contactPerson` | String | ❌ | ≤20 字符 | 可为空（null → ""） |
| 联系电话 | `contactPhone` | String | ❌ | 格式 `1[3-9]\d{9}` | 允许为空，非空才校验格式 |
| 地址 | `address` | String | ❌ | ≤200 字符 | - |
| 状态 | `status` | Integer | ✅ | 0=禁用 / 1=启用 | 默认 1；编辑时需回显 |
| 备注 | `remark` | String | ❌ | ≤200 字符 | - |

### 2.3 删除操作

| 场景 | 预期结果 | 后端 logic | 前端处理 |
|------|---------|-----------|---------|
| 供应商无关联采购记录 | 删除成功，`"供应商已删除"` | `supplierMapper.deleteById(id)` 软删除 | `ElMessage.success` + 刷新列表 |
| 供应商有关联采购记录 | **拦截**，`400 "该供应商存在关联采购记录，不可删除"` | `orderMapper.countBySupplierId(id) > 0` 抛异常 | `request.js` 拦截器自动 toast 错误消息 |

## 三、采购登记 — 字段级联调清单

### 3.1 登记表单 `ProcurementAddDTO`（POST `/api/procurement/add`）

| 前端字段 | 后端字段 | 类型 | 必填 | 校验规则 | 易错点 / 联调注意 |
|----------|---------|------|------|---------|-------------------|
| 采购单号 | `procurementNo` | String | ❌ | ≤50 字符 | 外部单据号，可为空 |
| 资产名称 | `assetName` | String | ✅ | 2-50 字符 | **前端+后端双校验** |
| 资产分类 | `category` | String | ✅ | 字典值（如 `IT_EQUIPMENT`） | 前端 `DictSelect` 组件自动加载 `asset_category` 字典；后端需校验供应商存在 |
| 规格型号 | `specification` | String | ❌ | ≤50 字符 | - |
| SN序列号 | `snNumber` | String | ❌ | ≤50 字符 | - |
| 采购数量 | `quantity` | Integer | ✅ | 1-100 | **前端总价 = 数量×单价 computed 实时计算**；后端 `totalAmount = quantity × unitPrice` |
| 单价 | `unitPrice` | BigDecimal | ✅ | >0，精度 2 | 后端精确到两位小数 |
| 总价 | - | - | 只读 | 自动计算 | 前端不应发送此字段，后端覆盖计算 |
| 采购日期 | `purchaseDate` | LocalDate | ✅ | ≤今天 | **后端校验 `purchaseDate.isAfter(LocalDate.now())` 则报错**；格式 `YYYY-MM-DD` |
| 供应商 | `supplierId` | Long | ✅ | 必须存在且 `status=1` | **后端校验供应商是否启用**；前端 `getAllEnabledSupplier` 只加载启用供应商 |
| 使用年限 | `usefulLife` | Integer | ✅ | 1-50 | 前端 `el-input-number :min="1" :max="50"` |
| 净残值率 | `residualRate` | BigDecimal | ❌ | 0-100，默认 5% | 后端默认 `5.00` |
| 所属部门 | `deptId` | Long | ✅ | 叶子节点 | 前端 `DeptTreeSelect` 组件 |
| 存放地点 | `location` | String | ✅ | 2-50 字符 | - |
| **验收状态** | `acceptStatus` | Integer | ✅ | 0=待验收 / 1=已验收 | **核心字段**，影响后续资产生成逻辑 |
| **验收日期** | `acceptDate` | LocalDate | **条件** | `acceptStatus=1` 时必填，≤今天 | **前端 v-if 条件显示**；后端判断 `acceptStatus==1 && acceptDate==null` 抛异常 |
| 备注 | `remark` | String | ❌ | ≤500 字符 | - |

### 3.2 验收状态关键业务路径

| 验收状态 | 前端提示 | 后端行为 | 联调关键点 |
|----------|---------|---------|-----------|
| **待验收（0）** | 保存后 → `"采购记录保存成功，验收后资产将自动入库"` | 仅写入 `proc_order` | 不触发资产生成，不获取分布式锁 |
| **已验收（1）** | 切换时 → `ElMessageBox.confirm("确认验收并入库？系统将自动生成{N}项资产记录")` | `@Transactional`：写入 `proc_order` → 获取锁 → 批量生成资产编码 → 批量插入 `asset_info` → 更新 `acceptStatus=2` → 清除缓存 | **分布式锁**：`eams:lock:procurement:accept:{id}` + `eams:lock:asset:code:{categoryShort}:{yyMM}` |
| | 成功后 → `"验收完成，已生成{N}项资产"` | `accept_status` 最终写入为 **2（已入库）**，非 1 | 资产编码格式 `AS-IT-2606-0001` |

## 四、采购记录 — 字段级联调清单

### 4.1 查询参数 `ProcurementQueryDTO`（GET `/api/procurement/record/list`）

| 前端字段 | 后端字段 | 类型 | 匹配方式 | 易错点 |

### 4.2 列表展示字段（后端瞬态 `@TableField(exist=false)` 字段）

| 表格列 | 来源字段 | 填充方式 | 联调注意 |

| 供应商名称 | `supplierName`（瞬态） | `fillProcurementNames()` 按 `supplierId` 批量查 `proc_supplier` | **批量查询优化**：一次查所有 supplier_id，避免 N+1 |

| 部门名称 | `deptName`（瞬态） | `fillProcurementNames()` 按 `deptId` 批量查 `sys_dept` | - |

| 值 | 状态 | 颜色 | `el-tag` type |
|----|------|------|--------------|
| 0 | 待验收 | 灰 | `info` |
| 1 | 已验收 | 橙 | `warning` |
| 2 | 已入库 | 绿 | `success` |
| 3 | 已取消 | 红 | `danger` |

### 4.4 操作按钮可见性

| 操作 | 显示条件 | 后端限制 |
|------|---------|---------|
| 详情 | 始终显示 | 无限制 |
| 编辑 | `acceptStatus < 2`（未入库） | 后端：`acceptStatus >= 2` 则 400 |
| 删除 | `acceptStatus < 1`（仅待验收） | 后端：`acceptStatus >= 1` 则 400 |

### 4.5 编辑表单（补充验收）

| 场景 | 前端行为 | 后端行为 | 联调关键点 |
|------|---------|---------|-----------|
| 待验收 → 已验收 | `watch acceptStatus` 弹出确认框 → 提交 | `triggerAcceptance=true` → 调用 `acceptAndGenerateAssets()` | **前端 `_originAcceptStatus` 记录原始值** |
| 已验收 → 待验收 | **禁用**（`:disabled="_originAcceptStatus>=1"`）| 后端报 `400 "已验收记录不能改为待验收"` | 防止回退 |
| 已验收 → 已验收 | 仅修改非验收字段 | 正常 update，不触发资产 | - |

## 五、核心业务规则校验点

### 5.1 分布式锁

| 锁 Key | 场景 | 等待时间 | 持有时间 | 失败提示 |
|--------|------|---------|---------|---------|
| `eams:lock:procurement:accept:{id}` | 验收入库 | 3s | 30s | `"操作繁忙，请稍后重试"` |
| `eams:lock:asset:code:{categoryShort}:{yyMM}` | 编码生成 | 3s | 30s | `"编码生成繁忙，请稍后重试"` |

### 5.2 数据一致性

| 规则 | 后端实现 | 联调验证方式 |
|------|---------|-------------|
| 验收事务原子性 | `@Transactional` 包裹 add/edit 全流程 | 验收入库中途异常 → proc_order 与 asset_info 均回滚 |
| 资产编码不重复 | 分布式锁 + MAX 查询 → 批量分配流水号 | 同一类别批量验收 N 条 → 编码连续无跳跃、不重复 |
| 供应商删除保护 | `countBySupplierId(id) > 0` 时拦截 | 有采购记录的供应商 → 400 |
| 已验收不可删除 | `acceptStatus >= 1` 时拦截 | 已验收/已入库记录 → 400 |
| 已入库不可编辑 | `acceptStatus >= 2` 时拦截 | 已入库记录 → 400 |

### 5.3 资产编码规则

| 组成部分 | 来源 | 示例 |
|----------|------|------|
| 前缀 | `AS-` | AS |
| 类别码 | `CATEGORY_SHORT` 映射 | IT / OF / PE / VE / OT |
| 年月 | `LocalDate.now()` yyMM | 2606 |
| 流水号 | DB MAX + 批量偏移 | 0001, 0002, 0003... |

### 5.4 权限控制

| 接口 | 角色 |
|------|------|
| `/procurement-supplier/*` | `ROLE_SUPER_ADMIN` / `ROLE_ASSET_ADMIN` |
| `/procurement/add` | `ROLE_SUPER_ADMIN` / `ROLE_ASSET_ADMIN` |
| `/procurement/edit` | `ROLE_SUPER_ADMIN` / `ROLE_ASSET_ADMIN` |
| `/procurement/delete/{id}` | `ROLE_SUPER_ADMIN` / `ROLE_ASSET_ADMIN` |
| `/procurement/record/list` | `ROLE_SUPER_ADMIN` / `ROLE_ASSET_ADMIN` / `ROLE_DEPT_ADMIN` |
| `/procurement/record/detail/{id}` | 同上 |

## 六、前端组件与后端数据映射易错点

### 6.1 DictSelect / Dict 字典

| 字典 Code | 前端组件 | 后端存储 |
|-----------|---------|---------|
| `asset_category` | `<DictSelect dictCode="asset_category">` | `proc_order.category` 存 `dict_value`（如 `IT_EQUIPMENT`） |

**注意**：后端 `fillProcurementNames()` 查 `sys_dict_item` 时 filter `dict_code='asset_category'` + `status=1` + `dict_value IN (categories)`。

### 6.2 DeptTreeSelect / 部门

| 前端组件 | 后端字段 | 注意 |
|----------|---------|------|
| `<DeptTreeSelect v-model="form.deptId">` | `deptId` (Long) | 存储部门 ID；后端不做叶子节点校验（与资产模块一致） |

### 6.3 DatePicker / 日期

| 字段 | 前端格式 | 后端接收 | 校验点 |
|------|---------|---------|--------|
| `purchaseDate` | `value-format="YYYY-MM-DD"` | `LocalDate` | 不能晚于今天 |
| `acceptDate` | `value-format="YYYY-MM-DD"` | `LocalDate` | 条件必填、不能晚于今天 |
| `dateRange`（查询） | `value-format="YYYY-MM-DD"` | `String beginDate, endDate` | 前端数组转两个字段 |

## 八、前后端差异重点核对

| # | 核对项 | 前端 | 后端 | 状态 |
|---|--------|------|------|------|
| 1 | 供应商编码在新增时可输入，编辑时只读 | `:disabled="isEdit"` | 编辑 DTO 不区分新增/编辑 | ✅ |
| 2 | 验收状态切换确认弹窗 | `ElMessageBox.confirm` 含资产数量提示 | 无弹窗 | ✅ |
| 3 | `acceptStatus` 前端传 0/1，后端最终写入 0/2 | 表单 radio 0/1 | 验收后 set to 2（已入库） | ✅ |
| 4 | 资产自动生成时不立即回显编码 | 列表无编码列 | 日志可查 | ⚠️ 可优化 |
| 5 | 编辑弹窗中已验收 → 待验收不可选 | `:disabled="_originAcceptStatus>=1"` | 后端报 400 | ✅ |
| 6 | 详情弹窗仅展示不编辑 | `<el-descriptions>` 只读 | 无额外限制 | ✅ |
| 7 | 前端请求 loading 状态 | `v-loading` + `submitting` | 无感知 | ✅ |
| 8 | 异常消息前端直接展示后端 message | `request.js` toast | `BusinessException` message | ✅ |
